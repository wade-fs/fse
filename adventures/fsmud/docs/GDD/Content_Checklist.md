# Content Checklist - 《源流福爾摩沙》地標與記憶開發品質檢核表 (Checklist)

為了確保每一位尋源者在踏入新地標 (Site) 時，能獲得最高品質、最一致且最具台灣風土深度的遊戲體驗，新開發的地標與其關聯記憶必須 100% 滿足以下檢核清單。

---

## 📋 地標 (Site) 開發品質檢核項目

### 1. 基礎設定與地理網絡
- [ ] **地標識別符格式**：ID 必須使用 `snake_case`（底線小寫），且為 `{settlement_id}_{site_name}` 格式。
- [ ] **聚落關聯聲明**：該地標 ID 必須已寫入所屬聚落 YAML (`/data/yaml/settlements/{settlement_id}.yaml`) 的 `sites` 清單中。
- [ ] **地理雙向通路**：至少與一個鄰近地標有 `connections` 宣告。
- [ ] **歷史保護區標記**：如果是重要古蹟（如天后宮、龍山寺），`is_heritage` 必須設為 `true`，以阻擋破壞性操作。

### 2. look 與描述渲染
- [ ] **基本描述 (base_description)**：至少有 2-3 行富有感官細節（如味道、聲音、光線）的優美文學描述。
- [ ] **抵達通知 (travel_arrive_text)**：玩家以 travel 移動至此時的迎面感官文字描述。
- [ ] **多時代別名 (names)**：若橫跨多時代，在 `names` 欄位中配置各時代的稱呼（如清領大正驿、現代民雄火車站）。

### 3. 動態 Reveal 浮現層與 NPC
- [ ] **條件浮現層設計**：每個地標必須至少擁有 1 個 Reveal Layer（如特定的職涯等級、天賦或已解鎖之記憶）。
- [ ] **NPC 投影與對話**：若有條件浮現之 NPC，其 YAML 對話樹必須包含對「地名」與「當前時代主要歷史事件」的問答樹。

### 4. 歷史記憶與共鳴 (Memory & Resonance)
- [ ] **專屬記憶碎片配置**：每個地標必須至少配置 1 個關聯的 `trigger_site` 記憶碎片 YAML 檔案。
- [ ] **自動化測試覆蓋**：必須建立對應的整合測試 `test_{site_name}.c`，模擬 travel、Reveal 條件判定與記憶自動解鎖。
