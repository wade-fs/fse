# 02. World Model - 世界實體架構模型 (World Model)

《源流福爾摩沙》的世界採用「聚落 (Settlement) - 地標 (Site) - 歷史層 (History Layer)」三層嵌套拓撲模型。

---

## 🏛️ 三層架構圖與定義

```text
Settlement (聚落，對應行政區，如民雄、台南)
  ├── 擁有全域六維數值（人口、文化、記憶、貿易、向心、凝聚）
  ├── 管理底下所有 Site 地標列表與拓撲
  │
  └── Site (地標/站點，如老街、車站、安平)
        ├── 繼承所屬 Settlement 的文明脈絡，無獨立六維數值
        ├── 具備地理連通性（Connections）與前置 Reveal Layers 浮現條件
        │
        └── History Layer (歷史層，如大正驛、荷治大員)
              ├── 定位：site_id + era_id 雙向綁定
              └── 描述與 NPC 依時代動態載入與重繪
```

---

## 📐 邊界案例判定指南
*   **Settlement**：必須對應真實的行政區劃（鄉/鎮/市/區），是動態存檔（`.o`）的唯一最小主體單位。
*   **Site**：代表不具備行政自治的特定街區、歷史地景或古蹟（如大稻埕屬於台北、安平屬於台南、八卦山屬於彰化）。玩家只能探索 Site，不可在 Site 中定居或建置聚落等級設施。
