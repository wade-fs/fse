# 遺忘浪潮與失源者共鳴儀式設計 (Oblivion & Resonance System)

本文件定義《源流福爾摩沙》的核心危機機制——**「遺忘浪潮」 (Oblivion Crisis)** 與解方 **「失源者共鳴儀式」 (Specter Resonance Ceremony)** 的設計規格，以指導後續開發。

---

## 🌊 1. 遺忘浪潮與記憶衰減機制 (Oblivion Decay)

歷史的流逝與無人述說，會導致聚落的歷史記憶逐漸模糊。此系統由 `/daemon/oblivion_d.c` 與 `/daemon/settlement_d.c` 協同驅動。

### ⏳ 記憶衰減規則 (Decay Rule)
1. **衰減 tick**：每 5 分鐘 (`300` 秒) 觸發一次全島聚落檢查。
2. **基礎衰減量**：
   - 聚落記憶值高於 0 時，基礎衰減 `1` 點記憶。
   - 若聚落記憶值高於 0 且**存在未被解除的失源者 (Specter)**，衰減量加倍，每次扣除 `2` 點記憶。
3. **安全線重置**：
   - 當世界完成時代推進 (`EraShifted`) 時，新時代的記憶力量會重置所有聚落記憶至安全線（若低於 `40` 則自動補足至 `40`），避免時代翻頁後立刻面臨遺忘危機。

### 🚨 遺忘危機警告 (Oblivion Crisis)
- **警示門檻 (Oblivion Warning)**：當聚落記憶值跌破 `OBLIVION_WARN` (`50`) 時，系統向該聚落地標的玩家發送「遺忘警訊」(`OblivionRising` 事件)。
- **危機門檻 (Oblivion Crisis)**：當聚落記憶值跌破 `OBLIVION_CRISIS` (`30`) 時，系統向**全伺服器線上玩家**廣播「遺忘危機」警告，提示該聚落記憶正被歷史遺忘，促使玩家前往救援。

## 👻 2. 失源者 (Specter) 的生成與互動

當聚落記憶值過低時，歷史缺口將實體化為「失源者」 (Specter)。

### 🧬 失源者生成規則
1. **觸發條件**：聚落記憶值低於 `OBLIVION_SPECTER` (`30`)，且該聚落目前無同型態的失源者。
2. **型態決定機制**：
   - 優先判定聚落的記憶子項目（如 Named Persons、Sites、Oral Tradition、History Events）何者最低。
   - 依最低項目或記憶數值生成對應型態的失源者實體，並移入該聚落的第一個歷史地標 (Site) 中。

### 🎭 四大失源者型態
| 型態常數 | 顯示名稱 | 核心描述 | 解除條件 (Resolve Conditions) |
| :--- | :--- | :--- | :--- |
| `SP_LOST_NAME` | **失名者** | 遺忘了自己名字的靈魂 | 玩家在此聚落已解鎖至少 `1` 則記憶 |
| `SP_LOST_PLACE` | **失鄉者** | 遺忘了故鄉、找不到歸處的靈魂 | 玩家已取得該聚落的「聚落踏印」 (`settlement:<id>`) |
| `SP_LOST_TONGUE`| **失語者** | 口不能言、無法訴說記憶的靈魂 | 玩家在此聚落已解鎖至少 `3` 則記憶 |
| `SP_LOST_HISTORY`| **失史者** | 被從歷史記載中抹除的靈魂 | 玩家文人職涯等級達 `2` 級以上 (`scholar` >= 2) |

### 💬 玩家與失源者的互動
- **察看 (look)**：察看失源者時，會顯示引導提示「*輸入 ask 失源者名 或 commune 可嘗試與它溝通*」。
- **詢問 (ask <specter> about <topic>)**：觸發失源者的專屬低語，並在玩家詢問時，嘗試藉由當前地標為媒介，在 `MEMORY_D` 中觸發未解鎖的記憶碎片。

## 🧘 3. 共鳴儀式與歷史共鳴 (Resonance Ceremony)

共鳴是玩家對抗遺忘的唯一武器，分為「被動共鳴」與「主動共鳴儀式」。

### 👥 被動歷史共鳴
- **觸發條件**：多名玩家（`2` 人以上）同時處於同一個歷史地標 (`Site`)，且該地標共鳴冷卻已過 (`120` 秒)。
- **共鳴強度**：參與玩家人數多寡決定強度（強度 `2` ~ `5`，上限為 `5`）。
- **共鳴效果**：
  1. **聚落記憶回升**：聚落記憶值回升 `強度 * 3` 點。
  2. **職涯點數獎勵**：在場的所有玩家各職涯均能獲得 `強度` 的修練點數。
  3. 觸發並廣播區域共鳴訊息。

### 🕯️ 主動共鳴儀式 (Commune Command)
玩家可於地標中輸入指令主動召喚歷史共鳴。
- **指令**：`commune`
- **使用資格**：玩家需對歷史有較深理解（文人等級 `2` 以上，或農人等級 `3` 以上）。
- **運作機制**：
  - 繞過人數判定，即使單人亦可對地標引導共鳴能量。
  - 主動共鳴觸發後，該地標將進入 `120` 秒共鳴冷卻。

### 🌟 解除失源者 (Resolve Specter)
當玩家滿足特定失源者的解除條件後：
1. 可透過與失源者共鳴或直接執行特定記憶解鎖，引導記憶分享給失源者。
2. 成功解除後，失源者將從場景中消散。
3. **危機解除獎勵**：該聚落記憶值立即回升 `10` 點，且全服廣播該聚落「記憶復甦」的好消息。

## 🛠️ 4. LPC 整合測試實作規劃

為了驗證上述機制的正確性，我們需要撰寫自動化整合測試，其要點如下：

1. **記憶衰減測試**：
   - 模擬調低聚落記憶值至 `OBLIVION_WARN` (`49`)，驗證是否觸發並廣播 `OblivionRising` 事件。
   - 進一步調低至 `OBLIVION_SPECTER` (`29`)，驗證 `settlement_d` 是否正確實體化對應的失源者 (如 `specter:lost_name`) 並移入地標中。

2. **失源者互動與解除測試**：
   - 驗證玩家對失源者進行 `respond_to_ask()` 後能讀取其回應並嘗試觸發記憶解鎖。
   - 驗證解除條件：未達標玩家解除失敗；已達標玩家呼叫 `resolve()` 成功消除失源者。
   - 驗證解除後，聚落記憶值正確回升 `10` 點，且 `oblivion_d` 的危機登記 (`crisis_registry`) 正確移除該失源者紀錄。

3. **共鳴儀式 (Commune) 測試**：
   - 驗證一般玩家使用 `commune` 指令因為職涯等級不足而被拒絕。
   - 調整玩家職涯等級，使其達標後成功發起 `commune`。
   - 驗證共鳴後聚落記憶值增加，玩家獲得修練點，且該 Site 進入冷卻狀態，無法在冷卻時間內重複引發共鳴。
