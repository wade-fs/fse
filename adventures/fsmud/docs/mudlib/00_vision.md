# 源流福爾摩沙 — 開發願景與哲學宣言 (MUDLib Vision & Manifesto)

## 文件定位

本文件是 《源流福爾摩沙》 MUDLib 開發的**憲章與最高指導原則**。

本文件旨在將小說 Canon 的文學內核（台灣歷史、地方誌、踏印、記憶與失源者），轉譯為具體的**遊戲機制設計哲學與工程開發鐵律**。不論是人類開發者或 AI 協同代理人，在編寫任何 LPC 物件、Daemon 或指令時，均須嚴格遵守本宣言。

---

## 核心定位：非征服型的世界模擬 MUD

傳統 MUD 的核心循環是「征服」：
```text
擊殺怪物 ➡️ 獲取經驗與裝備 ➡️ 角色等級數值膨脹 ➡️ 擊殺更強的怪物
```

《源流福爾摩沙》的核心循環是「理解與建構」：
```text
探索土地 ➡️ 收集歷史記憶與解鎖踏印 ➡️ 復興地方聚落 ➡️ 推進時代版本與歷史共振
```
玩家在此世界中扮演的是**文明的見證者與建設者**，而非征服者或掠奪者。

## 開發三大支柱 (The Three Pillars)

### 支柱一：以「踏印」取代等級 (Footprints over Levels)
本作完全揚棄傳統 RPG 的數值成長鏈。
- **工程約束**：LPC 玩家物件（`player.c`）中**禁止**定義 `level`、`exp`、`max_hp` 等隨時間無限增長的戰鬥數值。
- **成長指標**：玩家唯一的成長度量是「踏印（Footprints）」。
  - 地理、人情、技藝、聚落、時代、源流。
- **代碼設計原則**：所有技能解鎖、區域准入、任務權限，均須透過 `FOOTPRINT_D->has_footprint(me, footprint_id)` 進行判定。

### 支柱二：記憶是「第一類狀態」 (Memory as First-Class State)
在代碼中，記憶（Memory）不是唯讀的劇情文本或日誌，而是**系統運行的核心變數**。
- **動態影響**：聚落的記憶值（Memory %）決定了該地區的渲染色彩（前端 Emoji）、NPC 的態度語系，以及是否會產生「失源者」（遺忘的具象化）。
- **遺忘危機 (Oblivion)**：當聚落記憶值低於 30%，系統會動態開啟「遺忘狀態」，生成失源者（`Specter`）。
- **戰鬥哲學**：與失源者的戰鬥是「理解與救贖」的過程。擊敗失源者是為了「替它找回名字與歷史」，回饋是增加當地記憶值，而非掉落神裝。

### 支柱三：時間層疊的空間網格 (Time-Layered Space)
台灣是由多個歷史時期層疊而成的島嶼。
- **空間與時間繫結**：同一個地理坐標（如民雄網格），在不同版本時代擁有不同的 `world.c` 實體（如荒野聚落、乙未戰壕、現代車站）。
- **工程約束**：所有的網格地圖（Grid Map）必須預留 `era_id` 的動態加載接口，地圖邊界圍欄（Site Fences）需隨著歷史層的不同而自動變更。

## MUD 核心指令的語意轉譯

在編寫 MUD 常用指令時，必須將其與世界觀內核融合：

### 1. `look`（查看）
- **物理層面**：顯示 2D Minecraft 方塊地圖（草地、水池、木屋）。
- **語意層面**：呼叫 `SETTLEMENT_D` 取得當前坐標所屬聚落的文明指標（記憶度、文化度、商路）。
- **歷史層面**：若玩家擁有「島嶼記憶」天賦，`look` 可顯現空間中殘留的歷史痕跡。

### 2. `explore`（探索）
- **互動機制**：在網格中移動至特定坐標時使用。
- **工程觸發**：檢查是否跨越 `docs/mudlib/03_world_engine.md` 所定義的 Site 邊界。
- **回饋**：觸發 `FootprintGained` 事件，回饋當地聚落。

### 3. `build` / `place`（建造）
- **非純沙盒**：放置方塊不只是視覺上的堆疊。
- **社會回饋**：放置 `planks`（木板）或 `brick`（磚塊）代表參與聚落基礎建設，應透過 `EVENT_D` 增加聚落的「文化」與「凝聚力」。

## 開發者十誡 (The Ten Commandments)

1. **不可定義 RPG 等級**：違者扣除代碼優雅度。成長只看踏印。
2. **優先使用事件解耦**：模組間互動（例如獲得踏印影響聚落數值）一律使用 `EVENT_D->publish()`，嚴禁強耦合相互呼叫。
3. **動靜分離儲存**：靜態小說 canon 寫在 YAML；動態變動狀態寫在 JSON (`.o`)。存檔中不得複製 YAML 資料。
4. **尊重歷史保護區**：古蹟保護區內（`is_heritage == 1`）必須阻攔玩家的 `dig` 與 `place` 操作。
5. **多語系支持**：所有系統提示與敘述，均須呼叫 `select_lang` 或 `_t()`，支援中英雙語，適應星際網路定位。
6. **防範 Runtime 崩潰**：所有 Event 分發必須使用 `catch` 隔離，不能讓單一區域的錯誤導致全服中斷。
7. **無狀態服務 (Stateless Services)**：Daemon 只做協調與計算，不持有主體狀態，狀態由 Aggregate 自行維護。
8. **拒絕集權管理者**：沒有統管一切的 `SettlementManager.c`，只有各個獨立載入、動態填充的 `settlement` 實體。
9. **依據歷史層變更環境**：同一個 NPC 在不同歷史層應有不同的名字、對話與記憶。
10. **程式碼即史料**：保留代碼註解與文獻參照，所有 LPC 註解應指出對應的 Canon 文檔（如 `factions.md`, `timeline.md`）。
