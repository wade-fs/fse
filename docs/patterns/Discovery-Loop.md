# Pattern: Discovery Loop

> **一句話**：玩家的每一個行動，都是世界揭示自己的機會。

---

## 模式概述

Discovery Loop 是 FSE 最基礎的設計模式。

它描述的是：從行動到領悟的完整循環。

```
Act（行動）
    ↓
Consequence（後果，世界回應）
    ↓
Confusion（困惑，認知落差）
    ↓
Retry（重試，帶著新的假設）
    ↓
Discovery（領悟）
    ↓
Reveal（世界展示新的面貌）
    ↓
Act（新的行動，循環繼續）
```

---

## 為什麼這個循環有效

因為它模擬了人類真實的學習過程。

人不是因為被告知而理解。

人是因為：

**做了 → 失敗了 → 困惑了 → 重試了 → 突然懂了。**

困惑是這個循環最重要的部分。

沒有困惑，Discovery 就不會發生。

---

## 四個關鍵節點

### 1. Act（行動）

玩家對世界做任何事情都算。

移動、觀察、嘗試、寫程式、問 NPC。

關鍵是：**玩家必須主動**。

如果世界主動告訴玩家，循環就斷了。

---

### 2. Consequence（後果）

世界依照自己的規則回應。

不是「提示」，不是「警告」，而是**真實後果**：

```
走河谷 → 被發現 → 死
寫無窮迴圈 → 程式當掉
忘記補記憶值 → 失源者出現
```

後果必須是不可完全預期的。

如果玩家每次都能精準預測後果，

就不會有困惑，也不會有 Discovery。

---

### 3. Confusion（困惑）

**這是整個循環最重要的節點。**

困惑代表：玩家的心智模型與世界的規則之間，存在落差。

```
「我明明走得很遠，暴龍怎麼還是發現我了？」
「我只是讓它數到一百，為什麼程式當掉了？」
「我剛才才來過這裡，為什麼現在看起來不一樣？」
```

好的 World 設計，會主動製造這種落差。

---

### 4. Discovery（領悟）

玩家自己得出結論的那一刻。

不是世界告訴他，而是他自己說出：**「原來如此」**。

Discovery 之後，世界對他展示新的面貌（Reveal），

引發下一個循環的 Act。

---

## 三個 Adventure 的 Discovery Loop 範例

### Formosa

```
Act：走進老車站廣場
Consequence：失源者突然出現，記憶值下降
Confusion：「我剛剛什麼都沒做？」
Retry：查看廣場狀態，注意到「記憶值」指標
Discovery：「這個地方的記憶值不夠，需要主動踏查才能維持」
Reveal：廣場出現新的可探索區域
```

### Dinosaur World

```
Act：走在河谷裡，往暴龍方向移動
Consequence：暴龍衝過來，死亡
Confusion：「我明明走得很遠？」
Retry：第二次，注意到風向
Discovery：「暴龍靠嗅覺，不靠視覺，風向決定生死」
Reveal：逆風路線出現，可以進入高危區域邊緣
```

### Programming World

```
Act：while(true) { count++ }
Consequence：程式當掉，風扇狂轉
Confusion：「我只是要數到一百？」
Retry：加了條件，還是不對
Discovery：「迴圈需要終止條件，電腦不知道你心裡想的是一百」
Reveal：工具箱新增 break 積木
```

---

## 設計要點

**製造 Confusion 的方法：**

```
1. 讓後果出乎意料（但合乎世界規則）
2. 讓後果有延遲（不是立刻發生）
3. 讓後果有條件（不同情境，不同後果）
4. 讓後果不可完全還原（死了就是死了）
```

**常見的設計失誤：**

```
✗ 太快給提示，跳過 Confusion
✗ 後果太輕，玩家不在意（沒有 Confusion）
✗ 後果太重，玩家放棄（沒有 Retry）
✗ 後果完全可預期，玩家不感到驚訝
```

---

## 與其他模式的關係

- **Progressive Reveal**：Discovery Loop 驅動 Reveal 的更新
- **Hidden Knowledge**：Loop 的 Confusion 來自「知識被隱藏」
- **Environmental Teaching**：Loop 的 Consequence 透過環境傳遞
