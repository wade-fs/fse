# Pattern: Environmental Teaching

> **一句話**：讓世界本身成為老師，而不是讓 NPC 站出來解說。

---

## 模式概述

Environmental Teaching 描述的是：

**把知識內嵌在環境本身的行為裡，讓玩家透過與環境互動而理解。**

不是 NPC 說：「你知道嗎，暴龍其實靠嗅覺？」

而是：玩家走到暴龍附近，死了，開始思考為什麼。

環境是最誠實的老師：

它不解釋，不警告，只是**按照規則運作**。

玩家在環境的規則裡行動，自己得出結論。

---

## 環境教學 vs 直接教學

| | 直接教學 | 環境教學 |
|---|---|---|
| 知識傳遞方式 | NPC 說、說明文字、提示框 | 行動 → 後果 → 推斷 |
| 玩家角色 | 接收者 | 發現者 |
| 記憶留存 | 短暫（資訊） | 持久（經驗） |
| 知識內化 | 淺（知道） | 深（理解） |
| 失敗的意義 | 懲罰 | 資訊 |

---

## 環境傳遞知識的四種方式

### 1. 物理環境

地形、氣候、地質——這些本身就攜帶知識。

```
三疊紀的空氣：
  玩家跑兩步就喘
  → 不需要任何說明
  → 玩家自然推斷：這個時代的氧氣不夠

Formosa 的地景：
  打貓的地名、糖鐵的鐵軌痕跡、荒廢的廟宇
  → 每個地景都是歷史的殘留
  → 玩家透過觀察感受時間的流動
```

---

### 2. 生態行為

生物的行為模式本身是一套規則系統。

玩家觀察行為，推斷規則。

```
暴龍的行為：
  白天在開闊地巡邏（視覺捕獵）
  黃昏後行動慢下來（視力退化）
  鼻子總是朝風的方向（嗅覺導向）
  → 玩家透過觀察暴龍的行為，推斷出「靠嗅覺」

群居恐龍的行為：
  警報叫聲傳播方向
  集群保護的陣型
  → 玩家推斷：這個物種有複雜的社會行為
```

---

### 3. 系統狀態

世界的內部狀態，透過視覺/聽覺/觸覺呈現。

```
Programming World：
  記憶體用量顯示在環境裡（區域「壓力」視覺化）
  執行緒競爭時，系統顫抖
  迴圈沒有終止條件時，空間開始扭曲
  → 玩家透過感知系統狀態，理解程式行為

Formosa：
  失源者的密度代表記憶值衰退程度
  地景的清晰度代表歷史的保存狀態
  → 玩家透過世界的「健康狀態」感受時代的重量
```

---

### 4. 因果連鎖

玩家的行動引發連鎖後果，後果揭示規則。

```
Formosa：
  玩家踏查一個地點 → 記憶值上升 → 失源者減少 → 新的 NPC 出現
  這個連鎖本身就是「踏查能對抗遺忘」的教學

Programming World：
  玩家加了 break 積木 → 迴圈正常結束 → 系統穩定 → 新的節點開啟
  這個連鎖本身就是「break 解決無窮迴圈」的教學
```

## 設計要點

**問自己：**

```
1. 這個知識，能讓環境自己展示嗎？
   如果可以，就不要用 NPC 說出來。

2. 玩家在不理解這個知識的情況下，會遭遇什麼後果？
   後果必須是真實的、可感知的。

3. 後果本身能讓玩家推斷出知識嗎？
   後果要有足夠的資訊讓玩家思考。

4. 玩家有機會重複觀察，確認自己的推斷嗎？
   規則要一致，讓推斷有意義。
```

**常見的設計失誤：**

```
✗ 環境告訴了玩家太多（破壞發現感）
✗ 環境的反應太隨機，玩家無法推斷規則
✗ NPC 在旁邊解說：「你看，這就是因為...」（打斷環境教學）
✗ 知識隱藏得太深，玩家完全沒有線索（卡住）
```

## NPC 在 Environmental Teaching 中的角色

NPC 不是老師，但可以是：

```
提問者：「你有沒有注意到，暴龍最近都在往北走？」
（不給答案，只是引導玩家注意）

見證者：「上次有人走那條路，沒有回來。」
（暗示後果，不說原因）

學習者：「我也不知道為什麼，你如果搞清楚了告訴我。」
（和玩家一起探索，強化「這是可以發現的」）
```

NPC 說出答案 = Environmental Teaching 失敗。

NPC 提出問題 = Environmental Teaching 成功。

---

## 與其他模式的關係

- **Discovery Loop**：環境觸發 Consequence，啟動 Discovery Loop
- **Hidden Knowledge**：Environmental Teaching 是 Hidden Knowledge 的主要載體
- **Progressive Reveal**：隨著玩家理解加深，環境展示的細節也增加
