# Pattern: Hidden Knowledge

> **一句話**：知識不寫在說明書上，它藏在世界的運作規則裡。

---

## 模式概述

Hidden Knowledge 描述的是：

**世界的規則，只能透過行動來發現，不能直接被告知。**

這不是「故意隱藏資訊讓玩家猜謎」，

而是：**知識以情境的形式存在，而不是以文字的形式存在。**

```
❌ 錯誤的方式：
   說明文字：「注意！三疊紀氧氣只有 15%，跑步會消耗更多體力。」
   玩家：點頭，五分鐘後忘了。

✓ 正確的方式：
   玩家跑了兩步就喘。
   玩家：「為什麼？」
   （後來自己發現：氧氣只有 15%）
   玩家：永遠記得。
```

---

## 為什麼要隱藏知識

因為「被告知的知識」和「被發現的知識」，在記憶中的留存方式完全不同。

被告知的：資訊。可以被忘記。

被發現的：經驗。內建在行動記憶裡。

Hidden Knowledge 確保知識只能透過**體驗**獲得，

這樣它才會真正被內化。

---

## 三種隱藏方式

### 1. 規則隱藏（Rule Hiding）

世界的因果規則，不寫出來。

玩家只能透過行動推斷規則的存在。

```
Formosa：
  規則：失源者在記憶值低於閾值時出現
  隱藏：不告訴玩家「記憶值低於 X 時失源者出現」
  發現：玩家在忽略了某個地點一段時間後，突然遭遇失源者

Dinosaur World：
  規則：暴龍靠嗅覺，不靠視覺
  隱藏：不在說明書裡寫「暴龍靠嗅覺」
  發現：玩家走得很遠還是被發現，開始思考為什麼
```

---

### 2. 前提隱藏（Prerequisite Hiding）

某些事物的存在，本身就是隱藏的。

玩家不知道「還有什麼沒發現」。

```
Formosa：
  玩家不知道老站長的存在
  直到他完成了嘉南平原的探索
  老站長才突然「浮現」在地點描述中

Programming World：
  玩家不知道有「break 積木」
  直到他理解了「迴圈需要終止條件」
  break 積木才出現在工具箱
```

這是 Progressive Reveal 的知識面。

---

### 3. 意義隱藏（Meaning Hiding）

資訊本身是可見的，但意義是隱藏的。

玩家要累積足夠的理解，才能讀出意義。

```
Dinosaur World：
  資訊可見：「一陣風從西南方吹來」（一開始只是背景）
  意義隱藏：這意味著你的氣味正在往暴龍方向傳播
  意義揭示：當玩家理解了「暴龍靠嗅覺」之後

Formosa：
  資訊可見：「這裡曾經叫做『打貓』」（看起來是無意義的地名）
  意義隱藏：「打貓」是平埔族語的音譯，這個地名本身就是歷史的痕跡
  意義揭示：當玩家完成西拉雅時代的探索之後
```

---

## 隱藏的邊界

Hidden Knowledge 不是「刻意刁難玩家」。

好的 Hidden Knowledge 有三個邊界：

```
1. 規則必須一致
   世界的規則不會因玩家而改變
   發現了規則，下次一定有效

2. 失敗必須有意義
   不是「隨機死亡」
   而是「有原因的失敗，玩家可以推斷原因」

3. 玩家必須有機會重試
   知識藏著，但發現的機會是開放的
   不是「只有一次機會」
```

---

## 設計要點

**問自己：**

```
1. 這個知識，如果直接告訴玩家，他會忘記嗎？
   如果「是」，就應該隱藏，讓他體驗。

2. 玩家遭遇這個知識的後果時，他能感到困惑嗎？
   困惑是發現的前提。

3. 玩家有沒有機會從失敗中推斷規則？
   世界的規則要一致，這樣推斷才有意義。
```

**常見的設計失誤：**

```
✗ 隱藏的知識沒有規律，玩家推斷不出來（變成運氣遊戲）
✗ 後果太模糊，玩家不知道自己錯在哪裡
✗ 隱藏的知識永遠都沒有辦法被發現（卡住了）
✗ 告訴玩家「還有 X 個知識沒發現」（破壞探索感）
```

---

## 與其他模式的關係

- **Discovery Loop**：Hidden Knowledge 是 Loop 中 Confusion 的來源
- **Progressive Reveal**：知識被發現後，Reveal 更新，讓玩家看見更多
- **Environmental Teaching**：環境本身就是隱藏知識的容器
