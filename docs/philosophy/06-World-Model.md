# 06 — World Model

> World 是 FSE 最核心的概念之一。
> 這份文件定義世界如何運作——不是程式如何實作，而是世界本身的邏輯。

---

## 世界不是背景

World 不是故事發生的地方。

World 是理解發生的地方。

差別在於：

```
故事的背景：玩家在背景前面，背景不回應玩家
FSE 的 World：玩家在世界裡，世界因玩家而改變
```

World 是老師，但它不開口說話。

它只是讓事情發生，讓玩家在後果中理解規則。

---

## World 的必要條件

一個 FSE 的 World，必須滿足三個條件：

### 條件一：因果成立

玩家的行動有真實後果，不是假的選擇。

```
✓ 玩家走河谷 → 被捕食者發現 → 死亡
✓ 玩家逆風行動 → 氣味傳播 → 被追蹤
✗ 玩家走河谷 → 「危險！請選擇其他路線」→ 安全離開
```

後果必須是真實的，不是提示。

---

### 條件二：知識內嵌於情境

知識不寫在說明書上。

知識藏在世界的運作規則裡。

```
✓ 玩家跑兩步就喘（氧氣 15%），玩家問「為什麼？」
✗ 說明文字：「注意！三疊紀氧氣只有 15%，跑步會消耗更多體力。」
```

知識必須是被體驗的，不是被告知的。

---

### 條件三：可以重複進入

```
第一次：不懂，死了
第二次：有點懂，但還是死了
第三次：真的懂了，活了
```

世界必須允許重複嘗試，

因為 Discovery 需要時間累積。

---

## World 的狀態模型

World 的任何一個時刻，都由三個維度描述：

```
世界狀態（World State）= (空間, 時間, 環境)
```

**空間維度**：玩家在哪個 Node

**時間維度**：World 的宏觀 Progress Stage

**環境維度**：當前正在作用的環境事件

這三個維度的組合，決定了：

- 玩家能看見什麼（Reveal Layer）
- 玩家能做什麼（可用行動）
- 玩家行動後的後果（Consequence）

---

## World 的因果循環

每一個 World 的核心，都是這個循環：

```
World State（當前狀態）
    ↓
Actor Action（玩家行動）
    ↓
Consequence（後果，不可完全預期）
    ↓
State Change（狀態改變）
    ↓
New Discovery Possibility（可能觸發領悟）
    ↓
New World State（新的當前狀態）
    ↓
（循環）
```

這個循環永遠在運作。

玩家只能在循環裡行動，不能跳出循環。

---

## World 的 Entropy（熵）

World 不會永遠靜止。

如果玩家不行動，World 也會變化。

這種自然的衰退，叫做 Entropy：

```
Formosa：歷史被遺忘，聚落記憶值自然衰退
Programming World：系統 Bug 累積，熵值上升
Dinosaur World：生態系統有自己的節律，環境持續變化
```

Entropy 讓 World 保持張力。

玩家不是在一個靜止的謎題前思考，

而是在一個持續變化的世界裡行動。

---

## World 的邊界

World 的範圍是有限的，不是無限的。

這個有限性，製造了約束。

約束製造了 Inquiry。

Inquiry 製造了 Discovery。

```
有限的世界 → 玩家遭遇邊界 → 困惑 → 探索 → 理解
```

如果世界沒有邊界，玩家就不會遭遇約束。

不遭遇約束，就不會有困惑。

沒有困惑，就沒有 Discovery。

---

## 三個 Adventure 的 World 對比

### Formosa

```
空間：台灣各地的 Site（地點）
時間：西拉雅時代 → 民主紀元（Era 推進）
環境：失源者出現、颱風、時代危機

知識內嵌方式：
  地景本身就是歷史（打貓→民雄，名字是時代的印記）
  NPC 在不同時代說不同的話
  失源者的出現讓遺忘變得有形

Entropy：
  聚落記憶值自然衰退
  玩家不行動，歷史就逐漸消失
```

### Dinosaur World

```
空間：三疊紀～白堊紀的生態區
時間：地質年代推進
環境：捕食者行為、氣候、晝夜循環

知識內嵌方式：
  空氣（跑兩步就喘，玩家問「為什麼？」）
  地形（河谷是捕食者走廊）
  氣味（風向決定你被不被發現）

Entropy：
  生態系統有自己的節律
  捕食者有自己的巡邏路線
  環境持續變化，玩家不能停在原地
```

### Programming World

```
空間：程式概念領域（Loop Zone / Variable Zone / etc.）
時間：學習階段（Stage 1 Sequence → Stage 2 Loop → ...）
環境：系統 Bug、記憶體狀態、執行緒競爭

知識內嵌方式：
  迴圈跑了就停不下來（直接體驗後果）
  變數消失了（作用域是可以感知的）
  程式當掉了（不是警告，是真實後果）

Entropy：
  系統 Bug 累積
  熵值上升後，新的阻礙出現
```

---

## World 設計的核心問題

一個 Adventure 設計師在設計 World 時，

應該問自己：

```
1. 玩家在這個世界裡行動，世界會真實回應嗎？
2. 世界的規則，藏在情境裡還是說明文字裡？
3. 玩家可以重複進入同一個地方，每次都有新的發現嗎？
4. 這個世界有 Entropy 嗎？它會自然衰退嗎？
5. 世界的邊界製造了什麼樣的約束？
```

如果以上五個問題都有好的答案，

這個 World 就能產生 Discovery。
