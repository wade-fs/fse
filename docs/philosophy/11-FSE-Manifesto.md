# 11 — FSE Manifesto

> Version: v1.0 (Frozen)
> 狀態：不可逆宣言

在 Flow State Engine (FSE) 中，我們不設計課程，我們建造世界。這份宣言定義了 FSE 在設計、開發與驗證時所依循的十條靈魂原則。

---

## 1. Curiosity before Knowledge (好奇先於知識)
玩家不是因為「知道」而行動，而是因為「行動」才逐漸知道。
系統的首要任務不是灌輸答案，而是製造一個讓玩家主動提出「為什麼」的情境。好奇心是知識發生的前提。

## 2. Worlds before Lessons (世界先於課程)
知識不應以規則列表、教學投影片或提示框的形式被塞給玩家，而是必須內嵌於世界運作的物理規律之中。讓世界本身成為老師，而不是讓 NPC 站出來說教。

## 3. Traversal before Instruction (歷經先於告知)
人類的真實理解不是被傳遞的（transferred），而是被經歷的（traversed）。經歷限制、經歷挫折、經歷困惑，最終在重試中領悟。

## 4. Discovery before Rewards (領悟先於獎勵)
世界觀改變的那一瞬間（Aha! moment）才是真正的 Progression。任何等級的提升、積分的累積或成就勳章，都不能也不該取代「原來如此」那秒心智重構所帶來的喜悅。

## 5. Players remember places more than facts (玩家記住的是地方，而不是事實)
人腦對地理位置與情境的記憶，遠比對孤立事實的記憶深刻。將歷史、科學或程式概念與特定的地景、Node 或情境（Situation）綁定，理解就會伴隨空間記憶永久留存。

## 6. Every system must preserve the desire to explore (系統必須維護探索的渴望)
任何設計如果破壞了玩家「再往前走一步」的意願，即是失敗。世界的邊界與約束是為了引導探索，而不是為了阻斷好奇心。

## 7. Confusion is a design primitive (困惑是設計原語)
沒有認知落差（Epistemic Gap），就不會有重構認知結構的契機。設計師必須學會克制「立刻給答案」的衝動，妥善保護玩家的困惑，因為那是 Discovery 發生的溫床。

## 8. Failure is high-density information (失敗是高密度的信息)
在 FSE 中，失敗（如程式崩潰、在生態區死亡）不是懲罰，而是世界在對玩家的假設進行「約束校正」。每一次失敗都應該帶有真實的 Consequence，為玩家調整下一次的行動提供線索。

## 9. Meaning is conditionally revealed (意義是條件式揭示的)
同一個地點在不同理解深度的玩家眼中，是不同的世界。資訊只有在玩家準備好去理解它的那一刻出現，才具有線索的意義，否則它只是破壞探索感的噪音。

## 10. Core is invariants, Runtime is temporary implementation (核心是不變量，執行期是暫時的實作)
FSE 真正留存的價值在於世界模型、理解的物理學與探索的哲學。無論底層是用 Go, Rust, LPC 還是 3D 引擎實作，這套關於人類如何透過探索世界來理解萬物的哲學永遠成立。
