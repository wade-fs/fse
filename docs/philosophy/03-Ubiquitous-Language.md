# Factor Space Engine
# v0.1 — Core Concepts

作者：Wade
日期：2026

---

## 一句話定義

> FSE 是一個情境式學習引擎。
> 玩家在世界裡行動，行動產生結果，結果觸發 Discovery。
> Discovery 不是獎勵，是世界觀改變的那一刻。

---

## 為什麼不是課堂

老師說：三疊紀氧氣比較低。

玩家：（五分鐘後忘了）

---

玩家跑了兩步就喘。

玩家：「為什麼？」

後來才知道：氧氣只有 15%。

玩家：（永遠記得）

---

這就是 Situated Learning（情境式學習）。

知識不是被告知的。

知識是被經歷的。

---

## 四個核心概念

```
World
Actor
Discovery
Path
```

這四個概念，在所有 Adventure 裡都存在。

形式不同，結構相同。

# 一、World（世界）

## 定義

知識存在的地方。

有情境，有因果，有時間。

World 不是背景。World 是老師。

但它不開口說話。

它只是讓事情發生。

---

## World 的必要條件

```
1. 因果成立
   玩家的行動有真實後果
   不是假的選擇

2. 知識內嵌於情境
   知識不寫在說明書上
   知識藏在世界的運作規則裡

3. 可以重複進入
   第一次不懂
   第二次有點懂
   第三次真的懂了
```

---

## Formosa 的 World

```
台灣。

從西拉雅時代到民主紀元。

知識內嵌於：
  地景（打貓→民雄，名字本身就是歷史）
  NPC 的反應（不同時代，同一個人說不同的話）
  失源者的出現（遺忘是有形的，不是比喻）
  時代推進（玩家的行動真的改變世界狀態）

因果：
  玩家探索老車站 → 糖鐵踏印 → 聚落記憶值上升
  記憶值夠高 → 時代翻頁 → 世界真的變了
```

---

## Dinosaur 的 World

```
三疊紀到白堊紀。

知識內嵌於：
  空氣（氧氣 15%，跑兩步就喘）
  地形（河谷是捕食者的走廊）
  氣味（風向決定你被不被發現）
  時間（某些恐龍只在黃昏出沒）

因果：
  玩家走河谷 → 被發現 → 死
  玩家第二次注意風向 → 活了
  玩家學會了：暴龍靠嗅覺，不靠視覺
```

---

## Programming 的 World

```
程式執行的世界。

知識內嵌於：
  迴圈跑了就停不下來
  變數消失了（作用域）
  程式當掉了（沒有終止條件）
  兩個程式搶同一個資源（競爭條件）

因果：
  小朋友寫了無窮迴圈 → 程式當掉
  小朋友：「原來迴圈要有終止條件」
  這個知識，他不會忘記
```

# 二、Actor（行動者）

## 定義

在 World 裡移動、行動、提問的人。

Actor 不是學生。

Actor 是用行動提問的人。

---

## Actor 的核心行為

```
Move    → 去一個地方
Observe → 看見世界的狀態
Act     → 對世界做某件事
Fail    → 承受後果
Try     → 再試一次，帶著上一次的記憶
```

注意：Fail 是必要的。

沒有失敗，就沒有真實的因果。

沒有真實的因果，就沒有 Discovery。

---

## Formosa 的 Actor

```
陳進（主角）。

天賦：island_memory（島嶼記憶）
  → 能感知歷史層的殘影
  → 其他玩家看不見的東西，他看得見

核心行為：
  travel 嘉義市      → Move
  look               → Observe（看見的東西取決於理解深度）
  explore 老車站      → Act
  （失源者出現）      → Fail（記憶值不夠，世界在衰退）
  記錄口述、修復古蹟   → Try（對抗遺忘）

Actor 的成長不是等級。
是踏印的累積。
是理解的深度。
```

---

## Dinosaur 的 Actor

```
一個穿越到中生代的觀察者。

沒有武器。
沒有超能力。
只有觀察和判斷。

核心行為：
  Move    → 在地形中移動（選擇路線）
  Observe → 注意風向、聲音、地面痕跡
  Act     → 躲避、繞行、等待
  Fail    → 被發現、被攻擊
  Try     → 帶著新知識再試

Actor 的成長：
  不是等級。
  是對這個生態系統的理解深度。
```

---

## Programming 的 Actor

```
一個在程式世界裡執行任務的角色。

任務失敗 = 程式的邏輯錯誤。

核心行為：
  Move    → 進入新的程式概念領域
  Observe → 看見程式的執行結果
  Act     → 寫一段邏輯、下一個指令
  Fail    → 程式崩潰、結果錯誤
  Try     → 修改邏輯再試

Actor 的成長：
  不是分數。
  是能解決更複雜問題的能力。
```

# 三、Discovery（發現／領悟）

## 定義

世界觀改變的那一刻。

不是獎勵。不是解鎖通知。

是玩家內心說「原來如此」的那一秒。

---

## Discovery 的特性

```
1. 不能被直接教
   說出來的 Discovery 不是 Discovery
   只是資訊

2. 必須由行動引發
   玩家做了什麼
   世界回應了什麼
   玩家才能領悟什麼

3. 有時間差
   Discovery 不一定在行動當下發生
   有時候是第三次失敗之後才突然懂了

4. 改變後續行動
   真正的 Discovery 會讓玩家下一步不一樣
   如果行為沒有改變，就還沒有真正 Discover
```

---

## Discovery 的觸發結構

```
情境（Situation）
    ↓
行動（Action）
    ↓
結果（Consequence）
    ↓
困惑（Confusion）← 這個很重要，不能跳過
    ↓
重試（Retry）
    ↓
Discovery
```

困惑是必要的中間狀態。

太快給答案，Discovery 就不發生了。

---

## Formosa 的 Discovery

```
舊名：Memory（記憶碎片）
真名：Discovery

範例一：
  情境：護送甘蔗到市集
  行動：走在鐵軌旁邊
  結果：糖鐵突然來了，NPC 喊「快閃！」
  困惑：「為什麼中午會有火車？」
  重試：下次提前問 NPC
  Discovery：「原來糖鐵有固定班次，這是日治時代的工業命脈」

範例二：
  情境：進入台北城門（乙未之夜）
  行動：試圖阻止辜顯榮出城
  結果：無論如何，城門都打開了
  困惑：「為什麼我什麼都做不了？」
  Discovery：「有些歷史，無法阻止。見證比行動更難。」

Discovery 在 Formosa 的現有實作：
  memory_d.c 管理觸發條件
  trigger_site：玩家到了哪裡
  前置條件：玩家理解到了什麼程度
  unlocked_memories：玩家的 Discovery 記錄
  → 這個系統的骨架是對的
  → 只是名字叫錯了
```

---

## Dinosaur 的 Discovery

```
範例一：
  情境：看見暴龍在遠處
  行動：「那麼大應該看不到我」→ 繼續走
  結果：暴龍衝過來，死了
  困惑：「我明明很遠？」
  重試：注意到風向在吹向暴龍
  Discovery：「暴龍靠嗅覺，不靠視覺。風向決定你的生死。」

範例二：
  情境：跑步橫越平原
  行動：全速跑
  結果：兩步就喘，停下來
  困惑：「我平常跑步不會這樣？」
  Discovery：「三疊紀氧氣只有 15%，這個星球還沒準備好讓哺乳類呼吸」
```

---

## Programming 的 Discovery

```
範例一：
  情境：寫一個「數到一百」的程式
  行動：while(true) { count++ }
  結果：程式當掉，電腦風扇狂轉
  困惑：「為什麼？我只是要數到一百？」
  重試：加了 if count == 100: break
  Discovery：「迴圈需要終止條件，電腦不知道你心裡想的是一百」

範例二：
  情境：兩個任務同時進行
  行動：兩個程式同時讀寫同一個變數
  結果：數字亂掉了
  困惑：「我明明寫對了？」
  Discovery：「競爭條件。並行不等於同步。」
```

# 四、Path（路徑）

## 定義

Discovery 解鎖的新可能。

不是關卡解鎖。

是理解深了之後，世界對你展示新的面貌。

---

## Path 的特性

```
1. Discovery 驅動，不是等級驅動
   你理解了，路徑才開
   不是你等級夠了，路徑才開

2. 路徑是雙向的
   新的路徑讓你看見新的情境
   新的情境帶來新的 Discovery
   Discovery 再開新的路徑

3. 路徑可以是：
   新的地點（去了才能去）
   新的 NPC 願意跟你說話
   新的歷史層可以進入
   新的挑戰可以面對
   新的視角可以理解
```

---

## Path 的核心循環

```
Discovery
    ↓
Path 開啟
    ↓
新情境
    ↓
新行動
    ↓
新 Discovery
    ↓
（循環）
```

---

## Formosa 的 Path

```
現有實作名稱：reveal_layer（條件顯示層）

運作方式：
  玩家沒有任何踏印
    → 看見：民雄的基礎描述

  完成嘉南平原探索
    → 看見：老站長（NPC 浮現）

  完成老站長任務
    → 看見：糖鐵遺址的描述

  完成西來庵踏印
    → 看見：歷史層預感

  天賦 + v2.1 時代完成
    → 看見：1915年民雄的歷史層入口

  嘉南源流踏印
    → 看見：所有時代同時疊合

這個系統在 Formosa 已經完整實作。
reveal_layer.c 就是 Path 的引擎。
```

---

## Dinosaur 的 Path

```
玩家不知道風向的重要性
  → 只能在空曠地帶行動
  → 高風險區無法進入

Discovery：風向決定氣味傳播
  → 可以利用風向遮蔽氣味
  → 可以進入捕食者領域的邊緣
  → 可以觀察到新的生態行為

Discovery：暴龍黃昏後視力下降
  → 可以在黃昏後行動
  → 接近暴龍巢穴
  → 發現暴龍其實會護雛
  → 新的 Discovery
```

---

## Programming 的 Path

```
只知道順序執行
  → 只能寫線性程式
  → 無法解決需要判斷的問題

Discovery：條件判斷（if/else）
  → 可以寫有邏輯分支的程式
  → 遇到需要迴圈的問題

Discovery：迴圈需要終止條件
  → 可以寫有限迴圈
  → 遇到需要同時做多件事的問題

Discovery：並行與競爭條件
  → 可以寫多工程式
  → 每個 Discovery 開啟下一層的問題
```

# 對照表：FSE 概念 vs Formosa 現有實作

| FSE 概念 | Formosa 現有名稱 | 實作檔案 | 狀態 |
|---|---|---|---|
| World | 福爾摩沙（台灣） | settlement_d.c, site_d.c | ✓ 完整 |
| Actor | 玩家（陳進） | std/user.c, career_d.c | ✓ 完整 |
| Discovery | Memory（記憶碎片）| memory_d.c | ✓ 骨架完整，名字錯了 |
| Path | reveal_layer | std/reveal_layer.c | ✓ 完整 |
| Situation | Site + Era 組合 | std/site.c, timeline_d.c | ✓ 完整 |
| Consequence | 聚落六維變動 | settlement_d.c | ✓ 完整 |
| Confusion | 失源者、記憶衰退 | oblivion_d.c, specter.c | ✓ 完整 |
| Retry | 修復師職涯行動 | career_d.c, cmd_record.c | ✓ 完整 |

**結論：Formosa 已經是一個完整的 FSE 實例。**

它只是還沒有意識到自己是。

# FSE 的最小核心

如果要從 Formosa 抽出可被其他 Adventure 重用的部分，只有這五件事：

```
1. World
   知識內嵌於情境的容器
   有因果規則
   有時間軸

2. Actor
   用行動提問的存在
   成長 = 理解深度，不是等級數字

3. Discovery
   由行動+結果+困惑觸發的領悟
   不能被直接教
   有前置條件（需要先有某些理解）

4. Path
   Discovery 解鎖的新可能
   條件式顯示，不是等級式解鎖

5. EventBus
   World 內部的因果傳遞機制
   行動→事件→結果→新狀態
   各模組解耦，透過事件溝通
```

這五件事，就是 `fse/core/` 真正應該有的東西。

不多，不少。

# 下一步

Formosa 繼續做，不改架構，不改名字。

等第二個 Adventure 開始（Programming 或 Dinosaur），

自然知道 core 裡什麼是真正共用的。

那時候再提取。

提取出來的 core 會是真實需求驅動的，不是想像出來的。

---

**現在可以做的一件事：**

把 Formosa 裡的 `memory_d.c` 和相關 YAML，

在註解裡加一行：

```
# FSE concept: Discovery
# 這不是「記憶收集」，是「情境觸發的領悟」
# 觸發條件 = 情境 + 行動 + 前置理解
```

不改 code，只改認知。

這樣當你回頭看 Formosa 的時候，

你看見的是一個 FSE 的實例，

不只是一個 MUD。

區別兩件事：

- 模擬：
  - 認知形成的過程
  - 知識與行動之間的因果
  - 理解深度如何改變世界對你的展示

- 不模擬：
  - 歷史結果（Formosa 的歷史事件不可改變）
  - 完美的生態系統（Dinosaur 是簡化模型）
  - 真實的程式執行（Programming 是概念層的模擬）
