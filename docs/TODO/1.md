# PW (Programming World) 最小可行閉環設計
# Version: v0.1

> **「從無窮迴圈的崩潰中，學會終止條件的智慧。」**  
> 本文件旨在為第二個冒險 **PW（Programming World）** 建立一個最小可行性產品（MVP）閉環。我們將 FSE 的四個核心概念（World、Actor、Discovery、Path）與 PW 專屬的 AST 語意映射完美結合，以 `fsmud` 的結構為參照，實現一個極簡的沈浸式學習關卡。

## 📂 一、 PW 的最小可行目錄結構

參照 `fsmud` 龐大的結構，我們在 `adventures/pw/` 底下只抽取 FSE 核心運作所需的最小骨架，實施程式與資料分離：

```plaintext
adventures/pw/
├── mudlib/
│   ├── cmds/
│   │   └── player/
│   │       └── execute.c              # 接收前端 JSON AST 並交由 VM 執行的指令
│   ├── content/
│   │   ├── discoveries/               # 領悟定義 (對應 fsmud 的 memories)
│   │   │   └── loop_termination.yaml
│   │   ├── quests/                    # 任務與 AST 驗證標準
│   │   │   └── count_to_100.yaml
│   │   └── sites/                     # World 的實體地標
│   │       └── infinite_loop_swamp.c  # 無窮迴圈沼澤
│   └── std/
│       └── reveal_layer.c             # 控制 Blockly 積木是否顯示的過濾器
```

## ⚙️ 二、 最小閉環實作 (The Minimal Closed Loop)

這是一個完整的 FSE 情境式學習流程：  
`玩家行動` ➔ `失敗 (系統枯竭)` ➔ `困惑` ➔ `修正` ➔ `領悟 (Discovery)` ➔ `解鎖新路徑 (Path)`。

### 1. World (世界與情境)
知識內嵌於程式執行的世界中。玩家進入特定的地標，並面對當前的環境危機。

*   **檔案**：`content/sites/infinite_loop_swamp.c`
```c
// Site_ID: infinite_loop_swamp
#include <ansi.h>
inherit SITE;

void create() {
    set("name", "無窮迴圈沼澤");
    set("description", "這裡堆滿了永遠停不下來的廢棄進程。");
    // 當前環境維度狀態：記憶體洩漏警告
    set("active_world_events", ({ "memory_leak_warning" })); 
}

// 行動與後果方程式
int receive_execution(object player, mapping ast) {
    if (ast["type"] == "Loop" && !ast["has_break"]) {
        // Consequence: 記憶體枯竭，系統崩潰
        player->add_physical_state("memory", -20); 
        tell_object(player, HIR "程式當掉，電腦風扇狂轉！記憶體洩漏！\n" NOR);
        return 0; // Fail 必須發生
    }
    return 1;
}
```

### 2. Actor & AST Mapping (行動與語意映射)
在 PW 中，輸入媒介是「程式語言/抽象語法」。玩家無論用 Scratch 積木或 Python 送出邏輯，都會轉成 AST。

*   **檔案**：`content/quests/count_to_100.yaml`
```yaml
# 任務：數到一百
quest_id: count_to_100
location: infinite_loop_swamp

# Semantic Validation: 100% 程式與資料分離
expected_ast: 
  type: Loop
  condition: "count < 100"
  act: "count++"
```

---

### 3. Discovery (觸發領悟)
當玩家經歷過程式當掉的「困惑（Confusion）」，並成功加上終止條件（如 `if count == 100: break`），就會觸發領悟。

*   **檔案**：`content/discoveries/loop_termination.yaml`
```yaml
discovery_id: loop_termination
name: "迴圈的盡頭"
trigger_condition:
  site: infinite_loop_swamp
  required_action: "execute_valid_ast"
  required_failure_history: "memory_leak_crash" # 必須先經歷過當機

# 真正的 Discovery 是玩家內心的「原來如此」
description: "原來迴圈需要終止條件，電腦不知道你心裡想的是一百。"
```

---

### 4. Path & Reveal (解鎖新路徑與介面)
Path 不是等級解鎖，而是理解深了之後展現的新面貌。在 PW 中，這實作為前端工具箱（Toolbox）的條件式顯示。

*   **檔案**：`std/reveal_layer.c` (擷取邏輯)
```c
// Render(S, Player_Discoveries) -> Semantics
mapping get_toolbox_reveal(object player) {
    mapping toolbox = ([ "basic_sequence": 1 ]); 
    
    // 如果玩家擁有 "loop_termination" 這個 Discovery
    if (player->has_discovery("loop_termination")) {
        // 動態 Reveal [直到...] 迴圈積木 與 [中斷] 積木
        toolbox["conditional_loop_blocks"] = 1;
        toolbox["break_block"] = 1;
    }
    
    return toolbox;
}
```

## 🏁 三、 閉環運作總結

1.  **Move & Observe**：玩家進入 `infinite_loop_swamp`，看到 `memory_leak_warning`。目前工具箱只有基本積木，隱藏了複雜積木以防認知過載。
2.  **Act & Fail**：玩家寫出 `while(true) { count++ }`，指令被解析為 AST 傳給 VM。程式崩潰，引發後果（扣除記憶體 20M）。
3.  **Confusion & Try**：玩家產生困惑（*「為什麼？我只是要數到一百？」*）並嘗試修改邏輯。
4.  **Discovery**：加上終止條件後，系統比對 AST 符合 `count_to_100.yaml`，觸發 `loop_termination` 領悟。
5.  **Path**：`reveal_layer` 更新，玩家的 Web 編輯器正式解鎖 **[中斷] 積木**。這個理解將開啟下一層的新挑戰（如並行與競爭條件）。
