# PW (Programming World) 最小可行閉環設計與實作狀態
# Version: v1.0 (已實作)

> **「從無窮迴圈的崩潰中，學會終止條件的智慧。」**  
> 本文件旨在為第二個冒險 **PW（Programming World）** 建立並記錄一個最小可行性產品（MVP）閉環的實作狀態。我們將 FSE 的四個核心概念（World、Actor、Discovery、Path）與 PW 專屬的 AST 語意映射完美結合，並抽象出可共用的 FSE 引擎核心 (`runtime/`)。

---

## 📂 一、 實作完成的架構：引擎核心與資料分離

有別於早期將所有系統綁死在 `fsmud` 或特定冒險中的設計，我們現在已經成功將 **FSE 核心** 與 **Adventure 內容** 徹底抽離，並達成 100% 的程式與資料分離：

### 1. FSE 共用引擎 (Runtime Core)
核心系統位於 `runtime/services/`，負責所有 MUD 共用的情境式學習機制：
- `node_executor.c`：通用關卡執行判定引擎，負責讀取挑戰 (Challenges)、進行 AST 彈性比對，並處理成功與失敗 (包含 **Confusion 狀態**)。
- `factor_service.c`：通用核心因素 (Factor) 管理器，負責發布 **Discovery 領悟** 與前置條件檢查。
- `progress_manager.c`：多線並行進度管理器，負責驗證玩家是否滿足升階條件，並實作了隨時間衰退的 **Entropy (熵值)** 機制。
- `event_bus.c`：通用事件總線，解耦各個子系統。

### 2. 虛擬沙盒掛載 (Virtual Chroot)
由底層 Go 引擎 (`lpc-vm`) 支援，透過啟動參數 `-data` 與 `-runtime` 動態掛載實體資料夾：
```bash
./bin/fse -mudlib adventures/pw/mudlib -data data/pw -runtime runtime
```
這使得 `adventures/pw` 內完全不需使用軟連結 (symlinks)，且遊戲腳本永遠只需乾淨地讀寫 `/data/` 與 `/runtime/`。

## ⚙️ 二、 最小閉環實作 (The Minimal Closed Loop)

這是一個完整的 FSE 情境式學習流程，已經在 PW 中完整打通：  
`玩家行動` ➔ `失敗 (系統枯竭)` ➔ `困惑 (Confusion)` ➔ `修正` ➔ `領悟 (Discovery)` ➔ `解鎖新路徑 (Path)`。

### 1. World (世界與情境)
知識內嵌於程式執行的世界中。玩家進入特定的虛擬節點 (Node)，並面對當前的環境危機。
- **實作狀態 (✅ 已完成)**：PW 已經實作了基於 `virtual_object.c` 的通用資料驅動節點 (`/nodes/...`)。場景邏輯由 YAML 定義，並透過 `compile_object` 動態生成。
- **Entropy 機制 (✅ 已完成)**：玩家若長時間不互動，`progress_manager.c` 會觸發熵增，讓玩家感覺到情境連結變得模糊，迫使玩家「重複進入同一情境」以維持認知。

### 2. Actor & AST Mapping (行動與語意映射)
在 PW 中，輸入媒介是「程式語言/抽象語法」。玩家無論用 Scratch 積木或 Python 送出邏輯，都會轉成 AST。
- **實作狀態 (✅ 已完成)**：前端 Blockly 積木透過 WebSocket 傳送 AST，交由 `node_executor.c` 內的 `match_ast` 進行比對。
- 支援 `rule_based` 的強大彈性比對，甚至會自動往內解析 `Sequence` 節點以找到關鍵積木。

### 3. Confusion & Discovery (困惑與領悟)
當玩家送出錯誤的 AST 時，不僅會遭受後果，更會進入必要的「困惑」中間狀態，最終才能達成領悟。
- **實作狀態 (✅ 已完成)**：
  - 當 AST 不符預期，`node_executor.c` 會拋出 `"PlayerConfused"` 事件並給予玩家 `is_confused` 標籤，並在介面提示：`【 🌀 產生困惑 (Confusion) 】...`
  - 玩家修改邏輯後重試成功，解鎖 Factor 時，`factor_service.c` 會偵測到 `is_confused` 標籤，並印出敘述性過渡：`【 🌟 困惑 → 領悟 (Insight) 】 之前的疑惑瞬間解開了！`

### 4. Path & Reveal (解鎖新路徑與積木)
Discovery 解鎖的不是分數等級，而是對世界的全新認知，這展現在前端工具箱（Toolbox）的條件式顯示上。
- **實作狀態 (✅ 骨架完成)**：後端 Progression 系統 (`progress_manager.c`) 已經會根據解鎖的 Quests 與 Factors 自動推進玩家的 Stage。下一步是將這些 Stage 與 Factor 同步給前端，以動態 Reveal 新的 Blockly 積木分類。

---

## 🏁 三、 閉環運作總結 (現狀)

1.  **Move & Observe**：玩家進入 `/nodes/loop_swamp`。
2.  **Act & Fail**：玩家前端拉出一個沒有終止條件的迴圈，送出 AST 給 `execute` 指令。`node_executor.c` 解析失敗。
3.  **Confusion & Try**：系統扣除 `memory` 數值，印出崩潰警告，並賦予玩家 **Confusion** 狀態。玩家感到困惑並嘗試修改。
4.  **Discovery**：玩家加上 `Break` 積木後重試。`match_ast` 成功。`node_executor.c` 呼叫 `factor_service.c` 解鎖 `loop_termination` Factor，並印出「困惑 → 領悟」的完美過渡。
5.  **Path**：任務完成，`progress_manager.c` 檢測條件滿足，將玩家推進到下一個世界階段 (Stage)。前端將解鎖更進階的積木工具。

**結論**：PW 的最小可行閉環已經在底層引擎 (Go VM) 到 MUD 核心框架 (LPC Runtime) 完全實作並打通。未來 PC (史前文明) 等其他專案，也能無縫繼承這套完美的情境式學習引擎。
