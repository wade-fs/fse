# FSE TODO: 演進環境訊號作為 Input Medium (PC - 史前文明)

## 核心理念
目前 `pc` (Prehistoric Civilization) 仍然以傳統指令（如 `kill`, `look`）為主。為了完美契合 `Concept.md` 中對於「World 作為老師」與「Actor 透過觀察來行動」的定義，未來的核心互動模式應逐步廢除具象化的動作指令，轉而將「環境訊號 (Environmental Signals)」作為最主要的 Input Medium。

## 具體設計方向

### 1. 將指令解耦為「感官訊號 (Sensory Signals)」
未來的互動不再是「行動決定結果」，而是「感官收集與解讀」：
- **輸入端 (Input) 轉為感官聚焦**：玩家的指令不再是 `look` 或 `kill`，而是 `focus smell` (專注嗅覺)、`listen` (傾聽環境) 或是 `observe ground` (觀察地面痕跡)。
- **輸出端 (Output) 轉為環境參數**：世界（伺服器）不再直接給予確定的物件答案（例如：「這裡有一隻暴龍」），而是提供環境線索（例如：「一陣潮濕的南風吹來，帶著濃烈的腥味與微弱的低頻震動」）。玩家必須自行根據這些「環境訊號」來判斷危機。

### 2. 環境作為行動的催化劑 (Context-Aware Parsers)
環境與天氣狀態將直接影響訊號的收集與指令的有效性：
- **情境遮蔽**：如果處於逆風狀態，嗅覺相關的指令將無法獲得有效資訊。
- **動態懲罰**：在黃昏或夜晚，視覺指令將受到嚴格限制。除非玩家已經透過前面的行動或探索，解鎖了特定的 Factor（如 `night_vision` 或 `dusk_adaptation`），否則難以看清周圍。

### 3. 與現有 FSE 核心機制的完美結合
這套系統可以極好地利用現有的 Factor 與 Entropy 機制：
- **Factor 作為翻譯機**：
  - 初期：玩家感受到「微弱低頻震動」時感到困惑（觸發 Confusion 狀態），只能盲目試錯。
  - 後期：當解鎖了 `factor: rex_vibration` 後，同樣的訊號傳來，客戶端或伺服器可以藉由 Factor 的能力，直接將訊號「翻譯」為：「你感覺到暴龍正在接近」，將抽象訊號轉為具體認知。
- **Entropy (熵值) 作為環境變數**：
  - 記憶與感知的衰退：如果玩家長時間未與某種環境訊號互動，系統的 Entropy 機制會讓玩家對該訊號的敏感度降低。
  - 動態環境影響：環境本身也具有動態性（如天氣變化），訊號會隨著天氣（起霧、大雨）被扭曲或削弱，玩家必須根據當下環境動態調整其「感官聚焦」策略。

## 實作路徑 (Roadmap)
1. **重構 `pc/mudlib/std/user.c`**：
   - 替換現有的 `process_input` 系統。
   - 引入一個 `signal_parser`，負責接收玩家的感官指令，並根據玩家目前的 Factor 與環境狀態，過濾並回傳合適的描述。
2. **重構地標與生物物件 (Nodes / NPCs)**：
   - 物件不再只擁有固定的 `short` 或 `long` 描述。
   - 物件應該向外廣播「氣味」、「聲音」、「震動」等屬性。
3. **擴充 EventBus**：
   - 讓環境變數變化（如下雨）時，可以透過 EventBus 即時通知玩家，改變他們接收訊號的難度。
