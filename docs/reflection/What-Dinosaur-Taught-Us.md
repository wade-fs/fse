# Reflection: What Dinosaur Taught Us

> PC（Prehistoric Civilization）還沒有實作。
> 但光是「設計」它，就已經改變了 FSE 的思考方式。

---

## 為什麼一個未實作的 Adventure 能教我們東西

PC 的設計過程，逼我們回答了幾個 Formosa 和 PW 沒有逼我們回答的問題。

這些問題本身，就是 FSE 的進步。

---

## 恐龍教會我們的第一件事：觀察是行動

在 Formosa 和 PW 裡，

Actor 的行動都是「做某件事」：

```
Formosa：travel、explore、record
PW：execute（執行程式）
```

但 PC 的玩家，很多時候只是**看著**。

```
觀察暴龍的行為模式
等待風向改變
追蹤捕食者的巡邏路線
```

這些「觀察」應該觸發 Discovery 嗎？

答案：**應該。**

觀察是一種行動。

它消耗時間，承擔風險，並且可以改變玩家的理解。

這讓我們意識到：

**FSE 的 Actor 行動不只是「改變世界」，也可以是「感知世界」。**

這個理解改變了我們對 `node_executor` 的設計思路。

---

## 恐龍教會我們的第二件事：世界不需要等玩家

在 Formosa 裡，

如果玩家不行動，聚落記憶值會衰退。

這個衰退是「懲罰玩家不行動」的機制。

但在 PC 裡，

即使玩家不行動，世界也在演化：

```
捕食者在巡邏
獵物在遷徙
晝夜在交替
季節在轉換
```

這不是「懲罰」，而是**世界本身的生命**。

玩家的行動，是在一個活著的世界裡發生的，

而不是在一個等待玩家觸發的靜態場景裡。

這讓我們重新思考 FSE 的 Entropy 概念：

**Entropy 不只是「衰退」，而是「世界自己活著」。**

好的 World，不需要玩家才能讓它有意義。

玩家是世界的**訪客**，不是世界的**驅動力**。

---

## 恐龍教會我們的第三件事：死亡是信息

在大多數遊戲裡，死亡是懲罰。

在 PC 裡，死亡是**信息的最濃縮形式**。

```
第一次死：「暴龍發現了我」
第二次死：「走的是逆風路線」
第三次活了：「風向對了，它沒發現我」
Discovery：「風向決定嗅覺，嗅覺決定生死」
```

這讓我們重新定義 FSE 的「失敗」：

**失敗不是遊戲結束，而是信息密度最高的時刻。**

在 PC 的設計裡，「死亡」的後果是：

```
Knowledge Progress 保留（你學到的不會消失）
World Progress 部分重置（某些世界狀態回到初始）
Capability Progress 保留（你的理解仍然有效）
```

這個設計讓「死亡」從「失敗」變成「代價」——

有代價，但不是無意義的懲罰。

---

## 恐龍教會我們的第四件事：知識有層次

PC 的生態知識，不是平的。

它有層次：

```
Layer 1（感知）：
  風向、聲音、地面痕跡
  → 這些是可以直接觀察的

Layer 2（推斷）：
  暴龍靠嗅覺，不靠視覺
  → 這需要從 Layer 1 推斷出來

Layer 3（理解機制）：
  為什麼暴龍靠嗅覺？（演化的結果）
  → 這需要從 Layer 2 進一步推斷

Layer 4（系統理解）：
  整個生態系的捕食-獵物關係
  → 這需要理解許多 Layer 3 的知識之間的關係
```

這個層次結構，完美地對應了 FSE 的 Discovery 觸發條件：

**每個 Discovery 都需要前一層的 Factor 作為前提。**

PC 的設計，讓我們更清楚地看到：

Factor 的前置條件，不只是「你要先解鎖這個 Factor」，

而是「你要先真正理解這個層次，才能理解下一個層次」。

---

## 給 FSE 設計師的問題

PC 留給我們的問題：

```
1. 你的世界「活著」嗎？
   即使玩家不在，世界還是在動嗎？

2. 你的設計裡，死亡（或失敗）是信息嗎？
   還是只是懲罰？

3. 你的知識體系有層次嗎？
   玩家是從感知 → 推斷 → 理解機制 → 系統理解這樣進步的嗎？

4. 在你的世界裡，「什麼都不做，只是觀察」是有意義的行動嗎？
```
