# Reflection: Lessons from Formosa

> 這不是設計文件，而是學習紀錄。
> Formosa 在 FSE 框架命名之前就已存在，它教會了我們這個框架的輪廓。

---

## Formosa 先存在，FSE 後命名

我們沒有先設計好 FSE，再去做 Formosa。

順序是反的：

```
先有 Formosa
    ↓
在做 Formosa 的過程中，發現了一些反覆出現的問題
    ↓
這些問題逼出了 FSE 的概念
    ↓
FSE 命名之後，回頭看 Formosa
    ↓
發現 Formosa 已經是一個完整的 FSE 實例
    ↓
「它只是還沒有意識到自己是。」
```

這個反向的順序，是 FSE 設計最誠實的證明：

FSE 的概念，是從真實的 Adventure 設計中**提煉**出來的，

不是憑空想像再強加到 Adventure 上的。

---

## Formosa 教會我們的第一件事：名字很重要

Formosa 最初把「領悟」叫做「Memory（記憶碎片）」。

這個名字不對。

「記憶碎片」暗示：收集，累積，拼圖。

但 FSE 的 Discovery 不是收集。

它是：**世界觀改變的那一刻。**

當我們把「Memory」重新理解為「Discovery」之後，

整個設計的邏輯就清晰了：

```
Memory：玩家在收集什麼東西
Discovery：玩家在領悟什麼
```

這兩個名字，背後是兩種完全不同的設計哲學。

**教訓：名字決定了你怎麼思考系統。用錯誤的名字，會做出錯誤的設計。**

---

## Formosa 教會我們的第二件事：遺忘必須有形

最初，Formosa 的「遺忘」只是數字的衰退：

```
聚落記憶值：100 → 90 → 80 → ...
```

這個數字衰退，玩家感覺不到。

它只是介面上的一個數字。

後來，我們把遺忘具體化為「失源者（Specter）」：

```
記憶值不夠 → 失源者出現 → 玩家無法通行 → 玩家感受到遺忘的重量
```

失源者讓遺忘從「數字」變成「現實」。

玩家不是看著一個數字衰退而感到焦慮，

而是真實地遭遇了遺忘的後果。

**教訓：Confusion 必須在世界上有可感知的形式。數字不夠，要有形體。**

---

## Formosa 教會我們的第三件事：因果必須真實

Formosa 的歷史事件，**無論玩家做什麼，都不會改變**。

乙未之夜，城門終究會打開。

辜顯榮終究會出城。

玩家阻止不了。

這讓很多人覺得：「這樣玩家的行動沒有意義？」

但這正是 Formosa 最深刻的設計：

```
玩家以為可以改變歷史 → 發現改變不了 → 困惑 → Discovery：
「有些歷史，無法阻止。見證比行動更難。」
```

玩家的行動沒有改變歷史結果，

但改變了玩家對歷史的理解。

**教訓：因果的「真實」不一定是「玩家能控制結果」，而是「行動有不可逆的意義」。**

---

## Formosa 教會我們的第四件事：時間是最強的 Reveal 維度

Formosa 最獨特的設計，是把「歷史時代」作為 Reveal 的維度：

```
同一個地點（民雄），在不同時代：
  現代：普通的小鎮
  日治：製糖工廠的核心
  清治：「打貓」，平埔族聚落
  荷蘭：海岸貿易站的外圍
```

玩家不是在「學習」這些歷史，

而是在**不同時代的同一個地點之間移動**，

讓歷史自然地呈現。

這讓我們意識到：**Reveal 的維度不只有「玩家的理解深度」，還有「世界的時間維度」。**

---

## 給未來的 Adventure 設計師

如果你要做一個新的 Adventure，Formosa 留給你的問題是：

```
1. 你的世界裡，「遺忘」或「衰退」長什麼樣子？
   它有實體形式嗎？還是只是數字？

2. 你的世界有歷史嗎？
   玩家能在不同「時間層」之間移動嗎？

3. 你的世界裡，有些事情是玩家改變不了的嗎？
   這種「不可改變」，能製造什麼樣的 Discovery？

4. 你的世界的名字，有沒有誤導你的設計？
```
