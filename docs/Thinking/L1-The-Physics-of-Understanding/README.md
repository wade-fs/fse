Prologue — Before There Was FSE

Every engine begins with an idea.

FSE began with an observation.

每一個引擎，都源自一個想法。

FSE，卻源自一個觀察。

這本書不是從一個新的程式開始。

也不是從一個新的遊戲開始。

它開始於一個持續了三十多年的問題。

人，是如何真正理解世界的？

這個問題，並不是在設計 FSE 時才出現。

它早在第一個世界建立時，就已經存在。

只是當時，它還沒有名字。

一路走來，我建立過不同的世界。

有些是幻想世界。

有些是歷史世界。

有些是程式世界。

有些是史前世界。

它們擁有不同的角色、不同的規則、不同的故事。

但在一次又一次的設計中，我開始注意到一件事情。

玩家真正記住的，並不是我告訴他的知識。

而是他自己走過的路。

不是因為我解釋了歷史。

而是因為他站在歷史之中。

不是因為我講解了程式。

而是因為他親手讓程式失敗。

不是因為我介紹了遠古生態。

而是因為他在逆風中聞到了危險。

理解，從來不是答案的累積。

理解，是一次又一次探索之後，在心中逐漸形成的新世界。

於是，我開始意識到：

也許真正需要被設計的，不是教材。

不是任務。

甚至不是遊戲。

真正需要被設計的，是一個能夠孕育理解的世界。

一個世界，不需要不停地告訴玩家答案。

它只需要讓玩家願意再往前走一步。

因為真正的理解，總是在那一步之後發生。

很多人會把 FSE 看成一套 Runtime。

有人會把它看成一套 Adventure Engine。

有人會把它看成一種 MUD 架構，或是一套資料驅動框架。

這些都沒有錯。

但它們都不是 FSE 的起點。

FSE 的起點，不是程式。

而是一個更早出現的理解。

人不是因為得到答案而理解世界。

人是因為走進世界，而逐漸成為能理解它的人。

因此，FSE 並不是一項發明。

它更像是一項發現。

它並沒有創造理解。

它只是花了很長的時間，逐漸看清理解是如何發生的。

如果說，這本書有一個真正的起點，

那麼它不是 Factor Space Engine。

而是理解本身。

先有理解。

才有世界。

先有世界。

才有 FSE。

Runtime、Adventure、Factor、Discovery、Reveal Layer……

都只是這份理解的一種表達方式。

不同的人，可以用不同的語言實作它。

不同的平台，可以用不同的架構呈現它。

甚至有一天，當今天所有的程式語言都已經成為歷史，FSE 仍然可以存在。

因為它真正描述的，不是程式如何運作。

而是理解如何形成。

所以，閱讀這本書時，請不要急著尋找 API。

也不要急著尋找架構。

先問自己一個問題：

理解，是如何在一個世界裡誕生的？

如果這個問題開始在你的心中生長，

那麼，你已經走進了 FSE。

而這，也是這本書真正開始的地方。
