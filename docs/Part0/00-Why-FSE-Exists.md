# 第 0 章　為什麼會需要 FSE？

> 人很少記住別人告訴他的事情。  
> 人會記住自己曾經走過的地方。

這句話幾乎可以總結我過去三十多年建立世界的全部觀察。

我們花了許多時間設計課程、製作教材、建立測驗系統，卻一再發現：真正讓人改變的，往往不是那些被清楚講述的知識，而是在某個世界裡親身經歷的困惑、失敗、發現，以及那之後突然開闊的視野。

### 從「教」到「理解」

傳統的教育模式通常是：

老師 → 傳遞知識 → 學生 → 記憶 → 測驗

但現實中，人類的理解很少以這種線性方式發生。

我們真正記住的知識，幾乎都來自於：

- 在一個情境中親自行動
- 遭遇意料之外的結果
- 產生困惑
- 反覆嘗試
- 最終獲得「啊，原來如此」的瞬間

這就是 **情境式學習（Situated Learning）** 的核心。

FSE（Factor Space Engine）並不是要取代老師、教材或學校。  
它試圖做的，是創造一個**值得探索的世界**，讓理解在這個世界中自然發生。

### FSE 的本質

FSE 不是遊戲引擎。  
FSE 不是教學框架。  
FSE 也不是單純的程式架構。

**FSE 是一套描述「理解如何透過探索而形成」的理論，以及將這套理論轉化為可運作系統的實作。**

它的核心信念只有一句：

> **理解不是被教的，而是透過在世界中探索、困惑、發現而自然形成的。**

### 為什麼需要這樣一個引擎？

因為我們發現：

- 當玩家「走過」一個無窮迴圈的崩潰後，他們永遠記得「迴圈需要終止條件」。
- 當玩家在三疊紀逆風奔跑而被發現後，他們永遠記得「暴龍靠嗅覺」。
- 當玩家在歷史現場目睹一件無法改變的事件後，他們對「歷史」的理解，遠比讀十本教科書都深刻。

這些理解，不是來自講述，而是來自**親身經歷**。

FSE 想要做的，就是系統性地創造這樣的經歷。

它提供了一個最小但完整的框架，讓不同領域的人（歷史、程式、科學、生態、哲學……）都能建造出「值得探索的世界」，並讓理解在其中自然湧現。

### 本書的目標

本書不是一本技術手冊，也不是一本教育理論專著。

它試圖同時做到三件事：

1. 對程式設計師：展示如何建造一個真正以「理解」為目標的系統。
2. 對教育工作者：提供一種新的思考角度——世界本身可以成為老師。
3. 對所有思考者：探討「理解」這件事的本質，以及我們如何更好地支持它的發生。
